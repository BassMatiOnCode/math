Equivalence

Reflexive, Symmetric, Transitive

Introduction

As we have seen, relation R on a set S is a subset of a cartesian product S ⨯ S

R={(a,b)∈S⨯S|a,bsatisfya condition}

If we select a condition that selects pairs of equivalent elements, we get an equivalence relation.

The simplest equivalence relation is the equality "=", but any attribute can be used to define equivalence. For example, if we deal with Lego bricks, equivalence could be defined as "the same number of studs", or "the same color".

An equivalence relation has three properties: Reflexivity, symmetry, and transitivity. And if a relation has these properties, it is an equivalence relation.

If a relation is reflexive, each element of the set is equivalent to itself.

If an element a is equivalent to an element b, the reverse is also true:

If element a is equivalent to element b, and element b is equivalent to element c, then element a is also equivalent to element c.

Equivalence Relation

Equivalence can mean a lot of things. For example, with a set of Lego bricks, we could say that they are equivalent if they have the same number of studs. Or the same color. Applying the filter condition on every possible combination of elements from the set divides the original set S into non-overlapping subsets with equivalent objects. These subsets are called partitions. Depending on the equivalence relation, we get different partitions called equivalence classes.

A valid equivalence relation must have reflexive, symmetric and transitive properties:

The reflexive property means that every element a of S is equivalent to itself. For example, if an equivalence relation demands that bricks have the same color, then the condition is fulfilled: A brick has always the same color as itself.

The symmetric property means that if an element a is equivalent to an element b, then the element b is also equivalend to the element a. Applied to the color equivalence relation, if a yellow brick is equivalent to another yellow brick, then the reverse is also true.

The transitive property means that if an element a is equivalent to an element b, and if that element b is equivalent to an element c, then a must be equivalent to element c. Applied to the color relation on Lego bricks, if a yellow brick is equivalent to a second yellow brick, and if that second brick is equivalent to a third yellow brick, then the first yellow brick is equivalent to the third yellow brick.

At this point we will introduce a shorthand notation for the equivalence relation:

(a,b)∈R ⇔ a~b (a,b)∈R ⇔ a≡b

Some authors use the tilde, other use the triple bar character. We will use the tilde here.

With this notation at hand, we can write the equivalence properties much clearer:

Reflexivity: a~a∀a∈S Symmetry: a~b⇒b~a Transitivity: a~b∧b~c⇒a~c

Example: Congruence (mod n) Relation

We can define an equivalence relation for integers that are separated by multiples of n. For example, if n was 4, these numbers would all be congruent (mod n):

{ … ,−3,1,5,9,13,17,… }

NOTE that the parentheses around mod n are necessary because the term applies to the relation, but not to b.

Example with the numbers negative 3 and positive 9:

−3−9 = −12 = (−3)(4) a−b = kn

So indeed minus 3 is equivalent to 9 (mod 4). To show that congruence mod n is indeed an equivalence relation, we must prove the three equivalence relationship conditions.

Let's start with the reflexive property. We must plug in an element a from ℤ into the congruence mod n relation formula, like we did above:

a−a = k n 0 = k n 0 = k 0 ∈ ℤ

On the right hand side, the difference a − a clearly is zero. If we then divide by n, we get that k is zero, and zero is an integer number. So the reflexive propert is proven.

Next comes the symmetric property. We assume that

a ≡ b(modn) a−b = k n b−a=−(a−b) = −(k n)=(−k) n

In the first row we assume that a is congruent to b modulo n, which implies per definition the second row, the difference of a and b is an integer multiple of n.

In the third row, we want to show that b is congruent to a modulo n. If we factor a minus 1 out, we get the second term, which has the assumption above inside the parentheses. So we can derive from the row above that the terms must be equal to the negative of the right side above, minus (k times n).

Using the associative law for integer mutliplication, we can see that indeed (b minus a) is an integer multiple of n, because minus k ∈ ℤ.

Now the third property, transitivity. Suppose a≡b(modn) and b≡c(modn). Then choose k, l, j ∈ ℤ, such that

a−b=k n and b−c=l n a−c = j n a−b+b−c = j n (a−b)+(b−c) = (k n)+(l n) a−b+b−c = (k+l)n a−c = j n

The first line contains the definitions for the congruence relations. The second line contains the definition of the claim. In the third line we added a zero term, minus b plus b to the left side, which gives us the definitions from row 1 in row 4. On the right side, we can pull the right sides from row 1 and replace the right side in row 3. Essentially, we have added the two assumptions from row 1.

On the left siden in row 5, we can drop the parentheses because of the associativity of addition. On the right side, we can factor out n. This brings us to row 6, where we dropped the zero term minus b plus b. Because k and l are integers and their sum j is also an integer, the proof is complete.

We have all three properties verified, so a congruence mod n is an equivalence relation.
